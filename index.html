Chapter II · Six exhibits

What’s in the window.

Six snapshots out of thirty. Click a release in the Changelog and the next launch pulls it automatically.

I

The Hub

Editors & Creators, by category.

Achievements, Talents, Titles, Spells, Quests, Factions, Creatures, Currencies — plus the Skill Line Manager, Custom Class Creator, and an M2 Viewer. Each tool is a sandboxed feature module; the landing is the table of contents.

  • DBC
  • MySQL
  • Sandboxed
  • 30 tools
II

Realm discovery

A server browser that edits realmlist.wtf so you don’t.

Discover, filter, and connect to 3.3.5a realms. Pin favourites, see latency, and swap realmlists on connect. Server operators can submit their own listing with a tagline, a flag, and a Discord invite.

  • Realmlist
  • Latency
  • Submissions
III

Content marketplace

In-app Shop, Stripe-backed.

Content packs, templates, and tool add-ons — versioned, licensed, and delivered over the wire. Payments run through Stripe; the app handles install, verification, and update notifications.

  • Stripe
  • Licensed
  • Auto-install
IV

Community

Creator uploads, front-of-house.

Submissions from the community sit beside the featured catalogue. Preview before install, rate after, and creators get paid via Stripe Connect when someone buys theirs.

  • Stripe Connect
  • Ratings
  • Preview
V

The Codex

Documentation where you’re already looking.

Every editor documented in-app, cross-linked to emulator references. No tab-switching to a wiki — the docs live next to the fields they describe, and search is offline-first.

  • Offline
  • Cross-linked
  • Searchable
VI

Release notes

A changelog that pages itself.

Searchable timeline of what shipped, broke, and got fixed — with realtime updates over Supabase. When a new build lands, the client knows; when you pin a tool to a version, it stays pinned.

  • Realtime
  • Version-pinned
  • Timestamped

Chapter III · Under the hood

Built for the shape of this data.

  • Renderer

    Vite · React 19 · TypeScript

    Hash-routed SPA inside Electron. Feature-scoped modules, not a framework.

  • Native

    Rust · NAPI-RS · wgpu

    DBC parsing and M2 rendering live in Rust for the parts that can’t be slow.

  • Data

    DBC · MySQL · Supabase

    Client DBCs read-local. World MySQL writes for the six tools that need them. Supabase for auth & realtime.

  • Cores

    TrinityCore · AzerothCore

    DBC work is core-agnostic; MySQL writers detect the schema and adjust.

  • 3D

    Babylon.js · Three.js

    M2 viewer + map editor run in the renderer. Heavy ops offloaded to web workers.

  • Commerce

    Stripe · Stripe Connect

    One-time, subscription, and creator payouts — all wired into the in-app Shop.
